# Модель безопасности Spider

## Назначение

Инструмент создан **исключительно для администрирования машин, которыми вы
владеете** (свой рабочий парк, свои серверы). Не использовать на чужих
устройствах без явного разрешения владельца.

## Меры защиты

### Транспорт
- **TLS** терминирует Caddy (автоматический Let's Encrypt, HSTS).
- **Прикладное шифрование** поверх TLS: X25519 ECDH → HKDF-SHA256 → AES-256-GCM
  на каждое сообщение. Это defence-in-depth: даже при компрометации сертификата
  или прокси трафик остаётся защищённым.

### Авторизация
- **Admin API**: постоянный `ADMIN_KEY` (минимум 16 символов), сравнение
  constant-time. Хранится в панели в `localStorage` (не в cookie).
- **Агенты**: `device_id` + ключ сессии (проверка криптослоем при каждой
  операции). Регистрация — только по одноразовому enrollment-токену.

### Идентификация устройств
- **Enrollment-токен вместо «отпечатка системы»**. Каждый устройство
  регистрируется явно, с подтверждением на машине. Надёжнее при апгрейде
  железа и точно отражает состав парка.

### Агент
- **Видимый системный сервис** (systemd unit / HKCU Run), не скрытый.
- **Предупреждение при первом запуске** с запросом подтверждения.
- **Подписанное автообновление**: ed25519-подпись проверяется перед установкой;
  без неё обновление не применяется.

### Сервер
- **Systemd-сэндбокс**: `ProtectSystem=strict`, `ProtectHome`, `NoNewPrivileges`,
  `PrivateTmp`, ограниченные `ReadWritePaths`.
- **Логи аудита**: все admin-действия (создание токенов, команды, удаления)
  записываются в `audit_log` и видны в панели.
- **SQLite** с `foreign_keys(on)`, каскадное удаление, WAL-режим.

### Секреты
- **Никогда не в репозитории**. Все секреты (SSH, ADMIN_KEY, ключи подписи) —
  только через GitHub Actions secrets. В коде — ссылки на env-переменные.
- `server.env` на сервере имеет права `0600`, владелец `root:spider`.

## Что НЕ делается (и почему)

| Требование ТЗ                | Реализация                    | Причина                              |
|------------------------------|-------------------------------|--------------------------------------|
| Отпечаток системы-дедупликатор | Enrollment-токен            | Надёжнее при апгрейде; точный учёт   |
| Скрытый autostart            | Видимый systemd/Run сервис    | Управляемость; обнаружимость         |
| Тихая подмена версии         | Подписанное обновление        | Защита от подмены бинаря             |
| Секреты в публичном репо     | GitHub Actions secrets        | Защита от компрометации              |

## Известные ограничения

- **ADMIN_KEY — статический токен**. На больших инсталляциях стоит ввести
  ротацию / refresh-токены (TODO).
- **Long-poll опрашивает очередь раз в 500мс**. Для сотен устройств оптимизировать
  через notify-канал SQLite или Redis pub/sub (TODO).
- **Ротация ключей сессии** не реализована — ключ выдаётся один раз при enroll.
  Для постквантовой/компрометации стоит добавить rekey (TODO).

## Ответственное использование

- Смените креды, которые могли попасть в чат при настройке.
- Регулярно проверяйте `audit_log` на подозрительные действия.
- Ограничьте сетевой доступ к серверу (firewall на порты 80/443).
- Используйте сильный `ADMIN_KEY` (32+ символа) и храните его в менеджере паролей.
